Experiment 1 – CRUD Operations using Mongoose

Aim: To perform create, read, update, and delete operations using Mongoose.

Approach: A product schema was created using Mongoose. CRUD operations were implemented using Express routes. Data was stored and retrieved from MongoDB, and validation was applied on fields.

Experiment 2 – Student Management System (MVC)

Aim: To build a student management system using MVC architecture.

Approach: The project was structured into Model, View, and Controller. Mongoose was used for the model, Express handled routing and controllers, and EJS was used for the view. CRUD operations were performed through the UI.

Experiment 3 – Nested Schema and Aggregation

Aim: To create a product catalog using nested schemas and perform aggregation.

Approach: Nested schemas were created for product variants and reviews. Aggregation was used to calculate average ratings. Custom methods were implemented to update stock, and indexing concepts were applied for optimization.

Conclusion: These experiments helped in understanding backend development using Node.js, Express, and MongoDB. Concepts like CRUD operations, MVC architecture, schema design, aggregation, and API development were learned.
